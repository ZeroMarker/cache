1. 指标维护

一个完整的指标包括以下字段：ID、编码、指标名称、执行代码、节点标志、是否使用global、维度、指标区间。

 

 

1.1.            基本维护

1、新增指标

点击【新增】按钮，在指标列表中自动生成一条空白的指标。选中空白指标，填写一条指标的相关属性，如下图，点击【保存】按钮即可。

 

 

注意：编码名称是唯一的，若新建的指标编码在指标列表中存在，会提示字段重复，创建失败。

2、删除、更新、查询指标

删除指标：选中指标，点击【删除】按钮，即可删除指标，如下图。

 

 

注意：删除指标时，会删除所有勾选的指标及这些指标的所有指标维度、模块与报表中数据集的配置。

更新指标：选中指标，更改相应信息后，点击【保存】按钮即可。

查询指标：填写查询条件，点击【查询】按钮即可。

3、指标的复制、粘贴

指标的复制功能用于快速创建指标，指标系统V4.4.0版本增加了复制指标时能够同时复制指标维度的功能，如下图。

 

 

选中某条指标记录，点击【复制】按钮，提示“复制成功”。

点击【新增】按钮，新增一条空白记录。

点击【粘贴】按钮，可将原有指标维度同时复制到相应文本框中，如下图。

 

 

1.2.            指标维度维护

一个指标可以关联多个维度，作为指标维度。选中指标，右键点击【指标维度维护】，即可弹出“指标维度维护”窗口，如下图。

 

 

 

在“指标维度维护”窗口中点击【新增】按钮，填写相关内容后保存即可增加指标维度，其中关联维度编码是公共基础维度。

同时，每一个维度也可以有多个维度属性，维度属性是用来进一步解析维度值的，因此维度属性拥有执行代码。在“维度维护”页面中，选中维度，右键点击【维护维度属性】按钮，进行维度属性维护（详见“6、维度维护”章节）。

 

1.3.            维护指标日志

选中指标，右键点击【维护指标日志】，即可弹出“指标日志维护”窗口，如下图。

 

 

在“指标日志维护”窗口中点击【新增】按钮，勾选相应的日志项，点击【保存】按钮即可。维护了日志的指标可在“指标日志”模块显示相应的指标日志。

“维度选择”界面显示的是一个树结构，用户可以从中选择指标的维度以及维度属性。点击【配置取数规则】按钮，会弹出一个指标选择界面，用户可以从中筛选需要的指标，如下图。

点击【过滤规则生成器】按钮，弹出“过滤规则生成”窗口，如下图。

 

 

添加过滤规则后，过滤显示符合条件的数据，如下图。

 

 

点击【实时与历史对比】按钮，会显示历史与实时数据。如果两者不一致，对应的一栏会变红。

 

 

预览指标数据支持以pp，sl，s，c（上期，同期，汇总，区间自动升级数据）的方式查询，如下图。

 

 

当预览指标数据出现错误时，弹窗给出提示，如下图。

1.9.            指标维度模板

该功能用于对未设置指标维度的指标进行批量操作指标维度。选中指标，右键点击【指标维度模板】，弹出“指标维度维护模板”页面，如下图。

 

 

 

在“指标维度维护模板”页面中可对指标维度进行增加及删除，维护完成后，可通过右键点击【指标维度维护】进入“指标维度维护”窗口查看。

该功能可用于对未设置区间任务的指标进行批量区间任务操作。选中指标，右键点击【指标任务模板】，弹出“指标区间任务维护”页面，如下图。

 

 

 

在“指标区间任务维护”页面可新增、删除指标的区间任务。维护完成后，可通过【转到指标任务维护页面】查看指标任务。

注意：指标任务模板只能对未设置任务的指标操作。

1.11.       设置运算规则

选中计算类指标，右键点击【设置运算规则】，弹出“计算指标配置”页面，如下图。

 

 

 

“计算指标配置”页面可配置计算类指标的运算关系及汇总维度。通过双击列表中的指标将其添加到运算关系中。

 

 

页面中，“运算关系”中是用户配置的运算关系，其格式如下：

<指标代码1>+运算符+<指标代码2>+运算符+<指标代码n>

其中，指标代码N只能是在下面grid中出现的指标代码。这些供选择的指标代码与当前配置的计算指标有相同的基础维度。

运算符可以是加减乘除及括号。

汇总维度：计算指标的取值过程是先求出运算关系中每个指标的值，然后按某个或几个维度汇总，然后按照运算关系进行进行算术运算。因此，必须指定指标的汇总维度。

注意：只有取值方法为“计算指标”的指标可以设置运算规则。

取值方法分为：普通指标及计算指标。计算指标的值是其他指标通过算术运算得到的。普通指标就是除计算指标外的其他指标。

2. 任务设置和执行代码维护

任务设置和执行代码维护模块用于显示指标的执行代码以及配置指标的任务。指标现常用任务分为日、月、季、年4种任务，指标若想要生成数据，则需要在该模块配置并且激活指标任务。

每一个指标都可以做为一个独立的任务执行，执行任务后会根据指标的执行代码生成指标数据，并将生成的指标数据保存到指标数据表中。详细界面如下图。

 

 

在界面左侧区域查找到需要维护的指标，单击该指标。其相应的指标执行代码及指标区间任务显示在界面右侧相应区域中，如下图。

 

 

2.1.            指标执行代码维护

在“指标执行代码详细”区域填写或更改指标执行代码，点击【保存更新】按钮即可，如下图。

 

 

注：指标执行代码一般在“指标维护”页面新建指标时已经维护，如下图。

 

 

2.2.            指标任务维护

在“任务区间代码列表”区域进行指标区间任务的维护，如下图。

 

 

1、新增

点击【新增区间任务】按钮，系统会自动按照维护的指标区间和指标区间列表顺序生成任务区间，生成区间的顺序是从小到大的顺序（日、月、季、年的顺序）。新增完毕后点击【保存修改】按钮即完成任务的增加。

2、任务激活

当“是否激活”列为“Y”时，表示任务已经激活；为“N”时，表示任务未激活，此时，该任务虽然已经存在，但不会执行。点击【激活全部任务】按钮，可将该指标下的所有任务全部激活。

3、更新

点击任务记录中的“任务区间执行代码”列会弹出该任务对应的“指标区间任务执行代码选取”弹窗，如下图。

 

 

双击弹窗中的某条区间任务执行代码记录，该记录会更新到“任务区间执行代码”列中，如下图。

 

 

此时，“任务区间执行代码”列的左上角会出现红色三角标识，表明该处发生更改并未保存，点击【保存修改】按钮，左上角红色三角标识消失，任务区间执行代码的更新完成。

注：“指标区间任务执行代码选取”弹窗中的区间任务执行代码可在“任务执行代码配置”页面中以任务类型进行配置（详见“10.2任务执行代码配置”章节）。

点击任务记录中的“是否激活”列会显示激活状态下拉列表，如下图。

 

 

选择激活状态，并点击【保存修改】按钮，即可完成激活状态的更新。注意：当高区间为激活状态时，低区间不能为未激活状态，此时若点击【保存修改】按钮，会提示“业务操作失败！ 高区间与低区间激活状态配置异常”，并自动将未激活的低区间状态还原为激活状态，如下图。

 

 

 

4、删除

点击【顺序删除区间任务】按钮，可顺序删除区间任务，如下图。

 

 

区间任务是按照从大到小的顺序（年、季、月、日的顺序）删除的，这是因为父区间的任务数据，是通过其子区间的数据汇总而来的。

通过“指标导入和导出”界面可以选择需要的指标将其导出为xml格式，并且可以在新的指标环境中利用该模块的导入功能将该指标导入。导入功能分为三步：读入指标文件、导入前检查、导入指标。该功能极大的提高了指标的可移植性。详细界面如下图。

指标的导出有两种方式：1、在“指标维护”页面中导出指标（此方式详见“1.4导出指标”章节）；2、在“指标导入和导出”页面中导出指标。

在“指标导入和导出”页面中的左侧指标列表中勾选要导出的指标，点击【导出为XML文件】按钮，选择导出位置，点击【确定】按钮即可，如下图。

 

 

左侧指标列表支持按照指标代码、指标名称、指标区间等条件快速检索符合条件的指标。选择查找条件，在文本框中输入相应的信息，回车即可完成检索，如下图。

 

 

 

3.2.            指标导入

在“指标导入和导出”页面中的右侧区域中完成指标的导入，如下图。

 

 

选择导入的文件类型，点击【浏览】按钮，选择要导入的指标文件，点击【读入指标文件】按钮，系统提示“文件读入成功！”，并在页面右下方“导入指标列表”中显示将导入的所有指标，如下图。

 

 

点击【导入前检查】按钮，是判断要导入的指标与当前库中的指标有没有冲突，有冲突的指标会变为红色，表示库中已经有与要导入指标代码相同的指标，如下图。

4. 区间维护

区间是对一年日期的划分，目前指标系统实现的日期区间包括日、月、季、年4个区间，其中月为日的父区间、季为月的父区间，年为季的父区间。区间维护模块主要是用于对指标数据的管理，区间的设计实现了时间维度的分级，区间的主要作用是汇总指标数据，将低区间的指标数据汇总保存到高区间中，从而实现时间跨度较大的指标数据的查询工作能够很快完成。详细界面如下图。

 

 

每一个区间下都包括区间编码、区间名称、区间描述、直接父区间等。其中，“直接父区间”代表比该区间范围大的最近的区间，如：日的直接父区间是月，而不是季，年等。对于某一个区间来说，它的直接父区间的指标数据要从该区间汇总指标数据，这样，当查询数据区间跨度比较大时，可以提高查询速度。

1、新增区间

点击【新增】按钮，在区间列表中自动生成一条空白的记录。选中空白记录，填写区间相关信息，如下图，点击【保存】按钮即可。

 

 

注意：直接父区间必须按顺序维护，否则无法添加指标任务，且会影响指标数据的汇总，因此，区间维护模块默认开启强制维护区间顺序关系模式，即默认勾选“是否强制维护区间直接的顺序关系？”，当区间未按照顺序维护时，会提示“指标区间之间的父子关系维护不正确，要像链表一样”并将错误的直接父区间清除，如下图。

 

 

若不勾选强制模式，当区间未按照顺序维护时，仍会提示“指标区间之间的父子关系维护不正确，要像链表一样”，但会成功保存错误的直接父区间，如下图。

 

 

2、更新区间

选中某条区间记录，修改相关信息，点击【保存】按钮即可。注意：指标区间代码不能修改。

3、删除区间

在普通模式下，选中某条区间记录，点击【删除】按钮即可。注意：当为强制模式时，即勾选“是否强制维护区间直接的顺序关系？”，系统不允许删除区间，如下图。

 

 

4、维护区间维度属性

在“维度属性维护”窗口中可新增、删除维度属性。

5、导出到Excel文件

右键点击【导出到Excel文件】，可将全部区间信息导出，如下图。

 

 

 

 

5. 指标类型维护

指标类型维护模块可以根据指标的类别维护相应的类型，在新建指标时选择相应的类型进行保存，用户也可以根据该类型找到相应的指标。该模块的本质是对指标做了一个目录，方便指标管理。详细界面如下图。

 

 

1、新增指标类型

7. 模块与报表

模块与报表功能是为了对报表进行有效管理以及为项目实施提供便利而开发的一个功能。通常某些项目上的报表有几个或几十个，这些报表可以按照不同的特性进行分类，比如按照功能划分可分为：收入类报表、工作量类报表、病案类报表等等。模块与报表的功能之一就是对报表进行有序归类。

在进行项目实施（将某些报表配置到目标系统中）过程中，通常需要先确定这些报表需要用到哪些KPI，然后将这些最新的KPI导出后，导入到目标系统中。接着生成KPI数据，然后浏览数据是否正确生成，最后挂上菜单测试。配置了模块与报表功能后，这些操作步骤将得到简化。

该功能模块下分为模块、报表、数据集，多个数据集组成报表，多个报表组成模块，其中数据集保存有指标的取数规则和过滤规则，用户可以通过“模块：报表：数据集”的格式作为入参传入query，query可直接利用取数规则和过滤规则进行指标数据查询，该功能适用于对于取数规则和过滤规则比较固定的需求。简单的说，模块是具有相同特征的报表的集合；报表是数据集的集合；数据集是指标的集合。详细界面如下图。

10.    系统配置

系统配置模块内容涵盖过滤函数配置、任务执行代码配置、日志全局配置、指标任务全局配置、其他系统配置以及指标常见问题检查6个部分。详细界面如下图。

 

 

10.1.       过滤函数配置

该模块用于配置过滤规则生成器中用到的过滤函数。之前指标数据的过滤通常是在润乾报表中完成的，指标4.0版本之后引入了过滤函数的概念，使用过滤函数可以对指标数据进行各种条件的筛选，而不必在润乾报表中完成。过滤函数配置就是用来定义过滤函数的。详细界面如下图。

 

 

1、新增过滤函数

点击【新增】按钮，在过滤函数列表中自动生成一条空白的记录。选中空白记录，填写过滤函数相关信息，如下图，点击【保存】按钮即可。

 

 

其中，函数原型包括2种：INTERNAL（M语言支持的运算符）；CUSTOM（用户定义函数）。函数执行代码指调用过滤函数的代码。

注意：除了系统自带的可以识别的函数，如：<、=、>=等，对于用户定义的有执行代码的过滤函数，需要先在类文件中定义好函数名及实现代码，再进行新增。

2、修改过滤函数

选中某过滤函数记录，修改信息后，点击【保存】按钮即可。注意：不能修改过滤函数ID、函数原型、名称。

10.2.       任务执行代码配置

该模块用于保存指标任务执行代码。目前任务执行代码包括：指标、维度属性、任务、区间4种类型。其中指标、维度属性、区间类型目前只是在任务执行代码中可以被配置，不能被指标维护程序使用，只有任务类型的执行代码配置使用性较强。详细界面如下图。

 

 

1、新增任务执行代码

点击【新增】按钮，在任务执行代码列表中自动生成一条空白的记录。选中空白记录，填写任务执行代码相关信息，如下图，点击【保存】按钮即可。

 

 

注意：与“过滤函数配置”类似，有执行代码的任务，也需要先在相应文件中定义好函数及实现代码，再进行新增。

2、修改任务执行代码

选中某任务执行代码记录，修改信息后，点击【保存】按钮即可。注意：不能修改任务执行代码ID、类型、子类型。

10.3.       日志全局配置

该模块用于对所有指标的操作行为进行记录。针对全局配置概念的是局部配置，日志局部配置是对指定的某个或若干个指标进行日志配置（详见“8、日志配置”章节）；日志全局配置是对所有指标进行日志配置的方案。详细界面如下图。

 

 

    选择希望所有指标均记录的日志记录类型，点击【保存】按钮即可。

10.4.       指标任务全局配置

该模块主要包括2部分的配置：添加指标任务时，默认激活添加的指标任务；当某个指标任务出错时，任务挂起，同时将指标数据整个任务挂起。详细界面如下图。

 

 

1、添加指标任务时，默认激活添加的指标任务

指在“任务区间代码列表”（详见“2.2指标任务维护”章节）中新增区间任务时，新增的区间任务默认状态是否是激活（“Y”）状态。如果此处配置选择“否”，那么在新增区间任务时，新增的区间任务默认状态为未激活（“N”）状态，如下图。

 

 

2、当某个指标任务出错时，任务挂起，同时将指标数据整个任务挂起

指在每天凌晨滚指标任务时，如果某个指标发生错误，是否需要将整个任务都挂起。若为“否”，则出错的指标不再做任务数据，其他指标仍然要执行任务数据；若为“是”，则所有指标均不再执行做任务数据。
