定价类型使用说明

1.  定价信息使用背景

库存项的售价根据进价按照一定是规则(不加成,加成,不同的价位段不同的加成等)进行计算时,这样避免了我们人工进行繁琐的计算,可以避免由于计算错误导致售价错误的情况产生

2.  定价类型 和定价规则的含意

”定价类型”和”定价规则”这两个词的区别. 定价类型是物资(库存项) 维护的一个属性, 定价类型通过其设置的定价规则,结合进价计算售价来起作用.

定价类型可以有多个, 不同的物资根据加成规则的不同, 可维护不同的定价类型. 下面, 我们简单介绍

1.  我们维护一个定价类型, 命名为”物资定价类型”. (注意: 下限为不包括, 上限为包括)

2.  在定价规则界面, 维护3条定价规则, 将它们的”定价类型”指向刚刚维护的”物资定价类型”, 如下图

其中, 仅为”WZDJ01”设置了阶梯加成(仅在有明细勾选了”是否使用”选项的时候起作用), WZDJ02和WZDJ03均没有维护阶梯加成.

3.  在物资维护界面, 新建一个物资, 设置定价类型为”物资定价类型”, 我们来看售价的计算情况 (注: 后面的截图, 为了精简, 仅截取进价售价部分)

3.1  设置进价为300元, 售价为360元 (这里注意使用的是阶梯加成, 起作用的为”定价01”)

3.2  设置进价为600元, 售价为710元 (这里注意使用的是阶梯加成, 起作用为”定价01”, ”定价02”)

3.3  设置进价为1500元, 售价为1675元 (这里注意使用的是阶梯加成, 起作用为”定价01”, ”定价02”和”定价03”)

3.4  设置进价为3000元, 售价为3240元(这里注意加成率,加成额, 和最高加成率)

3.5  设置进价为6000元, 售价为6400元 (这里注意加成率,加成额, 没有超过最高加成率)

3.6  设置进价为10000元, 售价为10600元 (这里注意10000为WZDJ02的上限, WZDJ03的下限, 上限是包括的, 下限是包括的, 所以按照WZDJ02来计算)

3.7  设置进价为50000元, 售价为51500元 (这里注意没有超出最高加成额)

3.8  设置进价为100000元, 售价为102000元 (这里注意最高加成额)

4.  加成率, 加成额, 最高加成率, 最高加成额, 仅在非空且不为0时起作用.
小数规则, 旨在减少不必要的多位小数, 比如进价100.11元的物资, 按找”物资定价类型”计算的售价为120.132元, 是否小数位数方面, 可通过小数规则来设置. 若不需要, 在定价规则维护小数类型时, 选择一个明细为空的小数规则即可.

5.  目前根据进价和定价类型计算售价的处理, 在物资信息维护, 入库制单, 调价等处有使用. 其中入库制单和调价制单处需要通过参数设置来配置是否使用定价类型.

6.  部分项目因代码版本稍早, 该功能中的最高加成比等配置使用可能有些问题, 甚至存在不可使用的情况, 请参考最新代码或讨论, 谢谢^_^
